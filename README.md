| **缩写** | **全称**                          | **中文意思**   | **原因分析**                                                 |
| -------- | --------------------------------- | -------------- | ------------------------------------------------------------ |
| **WA**   | **W**rong **A**nswer              | **答案错误**   | 代码逻辑有漏洞，有些情况没考虑到（比如边界条件）。           |
| **TLE**  | **T**ime **L**imit **E**xceeded   | **超时**       | 算法太慢了，虽然结果对，但在这个测试用例上跑太久了（通常是死循环或者复杂度太高）。 |
| **MLE**  | **M**emory **L**imit **E**xceeded | **内存超限**   | 申请了太多空间（比如递归层数太深爆栈了，或者开了一个巨大的数组）。 |
| **RE**   | **R**untime **E**rror             | **运行时错误** | 程序崩溃了（比如除以0、数组越界、空指针）。                  |
| **CE**   | **C**ompilation **E**rror         | **编译错误**   | 语法写错了，连机器都读不懂（比如少个分号）。                 |
| **AC**   | **Ac**cepted                      | **成功**       |                                                              |
